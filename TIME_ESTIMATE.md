### 2. TTFT测量实验 🔄

**实验设计:**
- 测试用例: 10个对话输入
- 每次调用: System 1 (bridge) + System 2 (response)
- 指标收集: TTFT, System 2 latency, End-to-end

**时间估算:**
```

  - System 1 (0.5B): ~50-150ms
  - System 2 (7B): ~500ms-2s

```

**预计: 30秒 - 1分钟**

### 3. PNH准确率实验 ⏳

**实验设计:**
- 测试用例: 10个 (当前设置运行3个用于快速测试)
- 每个测试: 构建5-7轮对话历史 + 1次触发查询
- 每个测试调用: (对话轮数 × 2次LLM调用) + 1次最终调用

**时间估算:**
```
单个测试用例:
  - 对话历史构建: 5-7轮 × 2 LLM调用 × 500ms = 5-7秒
  - 触发查询: 1次 × 1.5秒 = 1.5秒
  - 评估和记录: ~0.5秒

单个测试总计: ~7-9秒

3个测试 (快速模式): ~21-27秒
10个测试 (完整模式): ~70-90秒
```

**预计: 30秒 - 2分钟**

## ⚠️ 实际运行时间影响因素

### 可能延长时间的因素:

1. **首次推理开销**
   - CUDA warmup可能需要额外10-20秒
   - **影响: +20秒**

2. **GPU内存分配**
   - 7B模型在4个GPU上分布可能有延迟
   - **影响: +10-30秒**

3. **对话轮数**
   - PNH实验需要构建真实对话历史
   - 如果每个测试用例有7轮对话，时间会显著增加
   - **影响: +2-5分钟**

4. **向量检索索引构建**
   - FAISS索引首次构建需要时间
   - **影响: +10-30秒**

5. **温度和其他生成参数**
   - temperature > 0 会增加生成时间
   - **影响: +20-50%**

## 💡 加速建议

如果需要更快的结果:

1. **减少测试次数**
   ```python
   # TTFT: 10次 → 5次
   self.run_ttft_tests(test_inputs, num_runs=5)
   
   # PNH: 10个 → 3个
   evaluator.run(limit=3)
   ```

2. **简化System 2调用**
   ```python
   # 减少max_new_tokens
   max_new_tokens=128  # 而不是256
   ```

3. **跳过System 2某些测试**
   ```python
   # 仅测量TTFT (System 1)
   # 不调用System 2
   ```

4. **使用8-bit量化**
   ```python
   load_in_8bit=True  # 节省显存，可能更快
   ```
